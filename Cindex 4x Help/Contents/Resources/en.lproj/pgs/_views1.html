Fully formatted view
 

To display the fully formatted index, choose Full Format from the View menu. There is also a corresponding item in the toolbar. The style of index and arrangement of entries are determined by choices you make from the Document menu.

In this view index entries are displayed in fully finished form, in whatever index style (indented, run-in, etc.) you have specified. Cindex suppresses the display of duplicate headings, gathers together and sorts page references from the same heading, and gathers and sorts cross-references from the same heading.

In the fully formatted view Cindex will hide deleted records and records that contain no text, and it will hide certain text in records—for example, phrases that you have inserted specially to control alphabetizing of entries. Cindex will display labeled records in their distinctive colors if you have enabled that (via a Preferences setting).

In the fully formatted index you can selectively hide records that carry particular labels (or no label at all). To set up hiding, choose Hide by Attribute… from the View menu.

You should always use a fully formatted view when printing the finished index or saving it for typesetting, but because this view obscures the structure of individual records you will probably find that during development of the index it is less useful than a draft formatted view.

When entries are displayed in a fully formatted view, Cindex can display different levels of heading in different fonts and type sizes, overriding the standard settings. When any standard setting has been overridden Cindex displays the standard font or size in the toolbar in red characters.
