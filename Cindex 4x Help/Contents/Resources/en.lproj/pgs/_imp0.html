Importing records
 

To import records from the following kinds of documents:

  • Any version of Cindex for Mac (filename extension .cdxf, or .ndxf)
  • Cindex for Windows version 3 and higher (filename extension .ucdx)

choose Open… from the File menu, or double-click the document's icon. From the opened document, drag (or copy) the records you want into the destination index.

To import records from an index created by Cindex for Windows version 2 or lower, the index should be in archive format. Choose Import from the File menu to import records into the destination index.

You should not use the archive format when moving a Cindex 4 index between Mac and Windows. The archive format does not allow representation of all Unicode characters.

Cindex can import records in several variants of a standard delimited format.

  • Plain Text — records contain plain text (not style or font information). Fields are separated by tab characters.
  • Sky Text — records are in the export format produced by Sky Index.
  • Macrex MBK — records are in the backup format used by the indexing program Macrex.
  • DOS Cindex data — records are in the legacy format used by Cindex for DOS and contain information about styles but not fonts. Fields may be separated by tab characters, or enclosed in quotation marks. Cindex assumes that documents with the filename extension '.dat' contain records in this format.
