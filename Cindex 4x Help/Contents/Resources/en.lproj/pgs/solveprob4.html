Preparing files for publishers
 

Here are answers to some frequently-asked questions about preparing files.

My editor has asked me for a Microsoft Word file. What file type do I save? RTF (Rich Text Format).

My editor has asked for sequences of tabs to be inserted to identify sub-headings in a RTF encoded file. Is she confusing tabs with the styles definitions that RTF provides? Perhaps. The leading indentation that Cindex provides for subheadings in a Rich Text Format file can be defined either as an attribute of the subheading, or as a sequence of characters (usually tabs). Which method Cindex uses by default is determined by settings you make by choosing Preferences (Export tab) from the Edit menu. When you use Save To... from the File menu to save an RTF file you can override the default setting.

My editor has told me that each index line in the file must not exceed 43 characters. How do I provide a file with this number of characters per line? You don't. When you save a formatted index in a file Cindex does not generally save line breaks, because you do not know the font or type size in which the index is to be set. If you want to view or print the index with the 43 character width limit, see here.

My editor has asked for a plain text file? What is this and how do I make it? A plain text file contains only the formatted text of entries; it does not contain information about type styles (e.g., boldface or italics) or fonts. To make such a file choose Save To... from the File menu, then in the sheet that Cindex displays choose Plain Text from the File Format pop-up menu. Note, however, that all information about type styles and fonts will be omitted from the file (it is retained in the original index). For this reason it is generally not a good idea to save indexes as plain text files.
