Setting record structure
 

Use the Record Structure panel (Document menu) to specify the structure and organization of index records.

•

To specify the maximum number of characters a record may contain (< 2000), enter the number at Maximum Characters in Record. If the index already contains records, the smallest number you can set is shown in the box immediately to the right.

•

To set the minimum number of fields (>= 2), choose from the Min pop-up menu.

To set the maximum number of fields (<= 16), choose from the Max pop-up menu.

If you set the minimum and maximum to be the same, records will contain a fixed number of fields.

By default, Cindex allows records to have variable numbers of fields up to five, and adds or removes fields automatically as you enter text in records.

•

To make every record have a special text field that immediately precedes the locator (regardless of how many normal subheadings the record contains), check Last subheading field is required. This field might be used for notes, or for storing URLs. If you wish, this field can be suppressed in formatted output, and ignored in sorting entries. This item is dimmed unless you have set the minimum number of fields to be at least 3.

When you have enabled the last subheading, Cindex treats that special heading as though it were at the deepest level permitted in the index, and applies the formatting associated with that level.

•

To set the name and other properties of and individual field choose it by its named position from the popup menu (for example 'First Field'). You can then set the field's name and other properties.

If you make settings when no index is open for work, these become the default ones that Cindex uses for new indexes when you create them.
