Checking records
 

As you store each record Cindex checks for certain typographical errors. It automatically removes any leading or trailing spaces in any field, and removes excess blank fields. If you have used parentheses or square brackets to enclose any part of a heading or any part of a page reference, Cindex checks for matching pairs. If it finds a mismatch, it will warn you with a beep and a message on the status line, and will highlight the field that contains the mismatched elements. If you really do want to leave the mismatched characters in a field, Cindex will accept the record when you make a second attempt to save it.

If you have enclosed text in < > or { } —special characters that Cindex uses to control the sorting of entries—Cindex checks that these occur in matched pairs in a field. If Cindex detects a mismatch, it will beep and highlight the field that contains the mismatched elements. If you make a another attempt to enter the record without having first corrected the error, Cindex will display an explicit warning message.

Cindex allows you to impose your own restrictions on the contents of individual fields of records—for example, you can stipulate that the locator field must contain only numerals. If you have imposed some restriction on the contents of a field, Cindex will warn you with a beep (and will highlight the offending field) if you try to save a record that contains forbidden characters or character sequences.

When first installed, Cindex allows you to add records that contain empty locator fields (if a record has an empty locator field, this usually means that you forgot to enter a reference). Cindex also allows you to enter a range of page numbers in which the second might be lower than the first (e.g., 95-93). You can set Cindex to warn you about (or forbid) the entry of a record that has an empty locator field or a malformed range of page numbers.

Cindex can check that each page reference you enter falls within specified limits. You can specify a value that no reference may exceed, and you can specify the greatest permitted span in a range of references.

You can set Cindex to warn you if any cross-reference you enter lacks a target—that is, no record in the index begins with the text of the cross-reference.
