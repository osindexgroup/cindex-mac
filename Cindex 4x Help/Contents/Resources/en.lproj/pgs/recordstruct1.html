Setting properties of record fields
 

Use the Record Structure panel (Document menu) to set the properties of each field of a record.

You can provide a name for each field in a record. Cindex displays field names in various pop-up menus and in the record window. You can also specify the maximum and minimum number of characters each field may contain, and a template that its contents must match before Cindex will save the record in the index.

Choose from the pop-up menu the field you want to work with (First Field for main heading, Second Field for subheading, etc. The last field is always reserved for locators.) You can make the following settings in the box:

•

To change the field name type a name in the Name box.

•

To specify a minimum number of characters the field must contain, enter the number at Min chars. If you set 0, Cindex sets no minimum value. You cannot set this value if the index contains records.

To specify a maximum number of characters a field can contain, enter the number in the Max chars. If you set 0, Cindex sets no maximum value.

If the index contains records, Cindex will display (on the right) the number of characters in the longest field at the current level. Cindex will not allow you to set Max chars to be smaller than this value.

•

To specify a template that text in a field must match, enter at Match a pattern to define the template. If you leave Match empty, Cindex accepts any text in a field.

If you have specified a pattern that should be matched by the text you enter, Cindex will check the field when you add or edit a record. If the text you enter does not match the pattern, what Cindex does depends on your Preferences settings.
