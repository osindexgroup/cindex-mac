Tagged-text documents
 

Many typesetting systems work with documents in which elements such as headings and subheadings carry identifying tags. The tags contain information about document structure, but little or no information about the attributes associated with tags (e.g., font and text style), so they say little about the document's final appearance. The final appearance depends on the attributes that the typesetter assigns to the tags.

Cindex comes equipped to make documents that are marked up using conventions derived from the Standard Generalized Markup Language (SGML) standard for constructing markup languages. Cindex provides for marking up using XML (Extensible Markup Language). XML is designed to handle Unicode text and is syntactically strict. Cindex also provides for markup using conventions such as HTML, and those recommended by the Association of American Publishers (AAP) and the University of Chicago Press (UCP). You can define your own sets of tags to suit your needs.

Use the Markup Tags panel (Tools menu) to view and edit sets of tags that Cindex uses to mark-up formatted documents.

Click the XML tab to work with tag sets for XML markup; click the SGML tab to work with tag sets for less strict SGML markup.

The pop-up menu shows all the available tag sets of the particular type. If the tag set is built-in to Cindex, you can view it (the rightmost button will be labeled View) but you cannot edit or delete it.

To view or edit an existing set of tags, choose from the pop-up menu then click View/Edit.

To duplicate an existing set of tags, choose from the pop-up menu then click Duplicate.

To create a new set of tags, click New.

When you used Save To… to save a tagged text document, Cindex offers as export formats the XML tag type currently selected via Markup Tags, and the SGML tag type currently selected via Markup tags.
