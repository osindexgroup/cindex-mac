Specifying the structure of page references
 

Use the Reference Syntax panel (Document menu) to specify how Cindex should recognize page references.

Settings for page references are made in the Page References box.

Cindex normally assumes that if several page references are in a single record, each will be separated from the next by a comma. Cindex also uses the comma when it concatenates references from several otherwise identical records. You can use a different separating character, but you must tell Cindex what that character is. Specify that character at Separator.

You will often indicate an inclusive range of pages (e.g., 155–158) by joining the references with a hyphen. Cindex needs to know how you denote an inclusive range, so that it can conflate or abbreviate references properly when it formats the index, and can adjust them properly when required. Specify the separating character at Connector. You can use any single ASCII character such as $ or @ that has no special meaning for Cindex. When Cindex produces the formatted index it will translate the connecting character into whatever character or sequence of characters you wish to have displayed and printed.

If you set Reference Syntax when no index is open for work, the settings become the default ones that Cindex uses for new indexes when you create them.
