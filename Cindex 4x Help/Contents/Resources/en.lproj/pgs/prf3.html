Settings for exporting formatted entries
 

Use the Preferences… panel (Cindex menu) to set preferences.

Settings you make under the Export tab determine how Cindex encodes information in documents that contain formatted entries for typesetting.

•

Choose from File Format how Cindex will represent the end of an entry (paragraph) in the file. Generally it will be safe to use the Windows setting for all files, regardless of whether they will be read on a Windows computer or a Macintosh.

•

Choose under RTF/Quark/InDesign/Plain text how you want Cindex to identify levels of subheading in the formatted text.

Choose style to include style definitions that programs such as Microsoft Word and QuarkXPress and InDesign can use to reproduce the layout you have worked with in Cindex.

Choose Tab if you want Cindex to insert one tab stop for each level of subheading.

Choose Other (and provide a single character) to make Cindex insert one such character for each level of subheading.

•

Check Include sorting information when embedding entries to make Cindex provide special sorting information when it embeds an entry in a Rich Text Format (RTF) document.

•

When you save a formatted index as Plain Text, Cindex can use Unicode (UTF-8) or Mac Roman character encoding. UTF-8 is recommended, but for backward compatiblity Mac Roman is the default. Click your choice at Encode Plain Text Documents.

When you save the formatted document you can override these default settings by clicking Options… in the Save To… panel.
