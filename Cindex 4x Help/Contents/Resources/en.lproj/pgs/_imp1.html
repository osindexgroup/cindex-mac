Working with index archives
 

An index archive is a compact copy of the index that contains information about the content of entries, including fonts and styles, but no information about the document format, or about how the index entries are alphabetized. An archive can be used by both Cindex for Mac and Cindex for Windows.

In Cindex 3 and later the archive format is deprecated, because it cannot represent all Unicode characters. You should use it only if you must share an index with someone who works with Cindex version 2 or earlier. Index documents (those with the .ucdx filename extension) are fully interchangeable betweeen Cindex for Mac and Cindex for Windows.

The recommended format for interchange with third party indexing software is XML records.

To save an index as an archive, choose Save To… from the File menu then from the File Format pop-up menu choose Cindex Archive. Cindex automatically provides the appropriate filename extension (.arc) when it saves an archive.

Cindex provides two ways to import records from archives: you can create a new index directly from an archive, or you can import the archived records into an existing index.

To add the contents of the archive to an existing index, choose Import… from the File menu, then select the archive you want.

To create a new index from an archive, choose Open… from the File menu, then select the archive by name from the displayed list. You can also do this through the Finder by double-clicking the archive’s icon. If no index with the name of the archive exists in the folder that contains the archive, Cindex will automatically create a new one with the name of the archive and will import the records. If an index with the name of the archive already exists, Cindex will present a panel through which you should choose the name and location of the index to be created.
