Settings for adding and editing records
 

Use the Preferences… panel (Cindex menu) to set preferences.

Settings you make under the Editing tab control how Cindex works when you are adding or editing records.

•

A draft view is often the best to work with when you add and edit records, because each record is always visible individually. To make Cindex switch to a draft view when you add or edit a record, check Switch to Draft View. If the original view was formatted, Cindex will revert to it when you finish editing.

•

When you add or edit a record, you might occasionally leave more than one space between words. To have Cindex remove these automatically, check Remove Extra Spaces.

•

When exchanging ('flipping') the contents of two record fields, Cindex can make a literal exchange, or can do it intelligently, adjusting lead letter case and moving articles and prepositions as appropriate to the content of the fields. To enable this intelligent exchange, check Smart Flip.

•

Cindex can generate the second part of a page range automatically when you type the character that connects the two parts of a range. For example, if a dash is the connecting character, and you typed 55- Cindex would complete the reference as 55-56 (you can, of course, then edit the number as required). Cindex does this only if the last recognized component of the last page reference in the locator field is a number in Arabic numerals. To enable this, check Complete Page Ranges.

•

When you are adding new entries (Edit -> New Record) Cindex normally displays the most recently-added records in record-number order. To make it show a new record immediately in its sorted position, check Track New Entries.

•

To make Cindex automatically re-use the locator (page reference) from the record you last-entered, check Carry Locators Forward.

•

To make Cindex use an existing heading in the index to complete (if it can) a heading you are typing, check Auto-Complete Entries.

If you want Cindex to match only the text of headings (not style and case) when it tries to complete a heading, check Ignores Style and Case.

If you want Cindex to display the existing entry from which it is taking a heading to use for completing a new one, check Track Source.

•

When you edit an existing record (Edit ->Edit Record) Cindex will normally adjust automatically the headings of any immediately following records whose headings originally matched headings you edited. If you don't want Cindex to do this, uncheck Propagate Changes. (You can override this setting for individual records by using a tool in the Record Window's toolbar.)

•

As you edit successive records in a series, Cindex adjusts the display in the main view to show your progress through the records. When you finish editing, Cindex normally leaves the display showing the last record you worked on. To make Cindex restore the display to show the first record you worked on, check Return to Entry Point.

•

When you add or change a label, Cindex by default does not mark the record as 'modified' (i.e., it does not update the date stamp and user identifier). To specify that a label change is considered a modification check Label Sets Change Flag.

•

Under Paste and Drop Text you can specify how Cindex deals with style and font information when pasting or dropping text that originates externally (e.g., from a PDF document). Choose whether Cindex accepts styles and fonts, styles only, or just plain text. The default is Styles Only. You can override the preference setting when actually pasting or dropping text.

•

Choose from Bad Locator how you Cindex should behave when you enter an empty or malformed locator, or one whose value is outside permissible limits.

•

Choose from Bad Cross-Ref how Cindex should behave when you enter in the locator field a cross-reference that has no valid target.

•

Choose from Mismatch to Template how you Cindex should behave when the text you have entered in records does not conform to the specification you have provided.

•

Choose from On Closing Window how how Cindex should behave if you close the Record Window with File->Close (COMMAND-W) or you click the close button, when the window contains edited text. . These settings do not apply when you complete editing with a keystroke: hitting COMMAND RETURN (or ENTER on the keypad) always saves the edits; hitting ESC always discards them.
