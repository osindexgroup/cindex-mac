Exporting index records
 

Cindex can save documents that contain just the contents of index records (without information about the style or layout of the index).

To save index records, choose Save To… from the File menu, then from the File Format pop-up menu select one of the following types of unformatted document:

Cindex XML records

Cindex can save records in an XML document. This is a plain text document, suitable for interchange with other indexing software. The XML document also includes information about fonts and styles used in entries.

Cindex archive

A Cindex archive is a document that contains all the index records (including information about fonts and styles used in entries). Because an archive cannot represent all the characters that might be used in an index, this format is deprecated except for interchange with earlier versions of Cindex.

Tab-delimited text

Cindex can save the text of index records (the contents of the record fields, including the locator field) in a form that is widely used by database management programs. In this form all the fields of an index record lie on a single line, each separated from the next by a ‘tab’ character. When imported into a database program, each field in the index record will occupy a different field in the database. The saved document contains only plain text; Cindex saves no information about fonts or styles. Cindex does not export text in { } used to control sorting, nor does it export any of the characters { } < > ~ \ unless they have been used literally in records.

Cindex by default uses UTF-8 (Unicode) character encoding. You can change the encoding to the Windows character set via Options… when the Save panel is displayed.

Cindex normally saves all of the index that is accessible in the current view (either all records, or a group). Cindex does not save empty records, and it normally does not save the text of deleted records. To modify this behavior, click Options in the Save panel before you save the records.
