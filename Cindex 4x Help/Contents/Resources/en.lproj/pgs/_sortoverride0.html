Overriding automatic sorting
 

Cindex provides two ways to override normal alphabetizing. If you know that you will always some text (e.g., St.) to be alphabetized as something else (e.g., Saint) you can set up automatic substitutions. To override alphabetizing in particular cases, you use special characters in records to control how Cindex sorts text.

To make Cindex ignore specified characters or words, enclose the characters or words in angle brackets. For example to make Cindex ignore 'Major' in "Hunter, Major William," you would set the text of the heading as:

Hunter, <Major >William

Cindex ignores characters in angle brackets during sorting, unless these are the only ones that distinguish entries. In that case the order of entries is determined by the characters in the brackets. The angle brackets themselves do not appear in the fully formatted display, though they do appear in the draft display.

In word-by-word alphabetizing, Cindex attends to spaces and hyphens between words. When you mark a word or passage of text to be ignored, you should always include within the angle brackets any trailing space or hyphen, as in the example above. If you leave a trailing space outside the brackets you may find that your entry is misplaced.

To make Cindex sort on text that is not a part of a record (e.g., a spelled-out number), enclose the text in braces. For example, to make Cindex sort "1066, The battle of Hastings" under 'ten-sixty-six' you would set the text of the heading as:

{ten sixty six}1066, The battle of Hastings

Notice that there is no space between the closing brace and 1066. Cindex uses the characters enclosed in braces to determine the alphabetical position of the record, but hides them and the braces when it displays the fully-formatted index (the braces and enclosed characters are visible in the draft view).

In the record-entry window Cindex provides toolbar buttons for enclosing selected text in < > or { }.

To use any of < > { } as a literal character in an entry, you must place \ immediately before it (e.g., \<).
