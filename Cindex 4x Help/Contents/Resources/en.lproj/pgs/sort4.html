Sorting locators
 

Use the Sort panel (Tools menu) to specify how Cindex should sort locators.

Settings in the Locators tab specify the how locators should be sorted.

Cindex evaluates page references under the following conditions:

  • When a single entry has multiple references, and these have to be ordered when the formatted index is displayed or printed.
  • When entries are ordered alphabetically, but are identical at all levels of heading that are being evaluated.
  • When you want entries ordered by page reference.

You can specify how Cindex should recognize locators (or parts of them) comprised of letters (as Roman numerals, spelled out months, or plain text), and what precedence it should assign to different classes of components. Through the Type Precedence list you specify whether or not Cindex will recognize a locator of a particular class, and (if it does recognize it) what precedence it should have. Cindex recognizes only segments corresponding to the classes that are checked with + marks. To check or uncheck a member of the list, double-click it. Cindex uses the order in which recognized classes appear in the list to determine the precedence of the components they represent. To change the precedence of a class drag it in the list.

If your locators have multiple parts, Cindex can ignore specified components and can evaluate the others in any order you choose. Use the Segment Order list to specify this. Cindex evaluates only reference segments associated with checked items in the list. Double-click an item to make Cindex evaluate (or ignore) the corresponding segment of a reference. The order in which the checked items appear in the list is the order in which Cindex evaluates their corresponding segments. Drag items to achieve the order you want.

Sometimes a text style (bold, italic, underline) is used to distinguish different types of references to the same page. Use the Style Precedence list to set the precedence of different text styles. Cindex uses the setting when the text style is the only attribute that distinguishes two page references (or the least significant parts of multi-part references).

If Cindex needs to use the value of a locator when positioning a record in the index (it generally does this only when positioning otherwise identical records), and the locator field contains more than one page reference, Cindex normally uses the lowest-valued one. To make Cindex use the highest-valued locator, uncheck Ascending Order. To make Cindex use the first locator, regardless of its value, uncheck Examine All.

When index entries contain bi-directional text (e.g., English and Arabic), the reading order of locators normally follows that of the script immediately preceding the locators. To force left-to-right reading order of locators, regardless of script, check Force left-right reading order.
